# 愤怒的小鸟游戏开发计划

## 项目信息

**GitHub 仓库**: https://github.com/abcdefg-maker/PhaserGameDemo-AngryBird-DevelopingWithClaudeCode

**Git 远程地址**: git@github.com:abcdefg-maker/PhaserGameDemo-AngryBird-DevelopingWithClaudeCode.git

**当前版本**: v0.5.0 - 小鸟系统（含特殊能力）

## 开发顺序

### 第一阶段：基础对象系统
1. ✅ **程序化图形生成系统** - 所有游戏对象的图形基础
   - 小鸟纹理（红鸟、黄鸟、炸弹鸟）
   - 方块纹理（木头、石头、玻璃，横竖方向）
   - 猪纹理（小、中、大）
   - 弹弓纹理
   - 地面纹理

2. ✅ **建筑方块系统**（木头、石头、玻璃）
   - 创建 Block 基类
   - 不同材质的物理属性（密度、摩擦力）
   - 耐久度系统

3. ✅ **猪系统**
   - 创建 Pig 类
   - 生命值系统
   - 不同大小的猪

### 第二阶段：核心玩法
4. ✅ **小鸟基类和全部小鸟类型**
   - 创建 Bird 基类
   - 红鸟实现（基础攻击）
   - 黄鸟（加速能力）
   - 炸弹鸟（爆炸能力）
   - 飞行物理和状态管理

5. **弹弓系统**
   - 拖拽交互
   - 力度计算
   - 轨迹预测
   - 发射小鸟

6. **碰撞伤害计算**
   - 碰撞检测
   - 伤害计算公式
   - 对象销毁逻辑

### 第三阶段：扩展功能
7. **关卡数据和管理器**
   - 关卡数据结构
   - LevelManager
   - 关卡加载

8. **相机跟随系统**
   - 跟随小鸟飞行
   - 平滑过渡

9. **分数和星级系统**
   - 分数计算
   - 三星评级
   - UI显示

## 技术要点

### 物理引擎配置
- 重力：{ x: 0, y: 1.5 }
- 启用休眠优化
- Matter.js 物理引擎

### 材质属性参考
- **木头**：密度低，容易破坏
- **石头**：密度高，难以破坏
- **玻璃**：密度中，易碎

## 开发原则
- 每个功能独立开发和测试
- 循序渐进，避免错误积累
- 及时提交 Git 版本

## 开发规范

### 文档同步更新规范
**重要：每次完成一个功能开发后，必须同步更新 README.md**

更新内容包括：
1. **更新版本号** - 在 README.md 顶部更新当前版本号
2. **更新已实现功能列表** - 将完成的功能从"待实现功能"移动到"已实现功能"
3. **更新待实现功能列表** - 移除已完成的项，保持列表准确
4. **更新项目结构** - 如果添加了新的文件或目录，更新项目结构图

这样做的目的：
- 保持文档与代码同步
- 让用户清楚了解项目当前进度
- 方便团队协作和项目管理
